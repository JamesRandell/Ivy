Ivy treats models as Data Dictionary files. They contain meta data about external data structures, validation, triggers and anything else that relates to a datasource. They are sometimes referred to in Ivy as Schemas

What makes up a Model

A model contains information relating to data stored in a database of some kind. This could be SQL, XML, Excel - it doesn't matter.

Each model will always contain the following:

  • fieldSpec handles fields and validation
  • tableSpec selects a specific table
  • databaseSpec custom connection to databases or file systems

Other uses

Form validation rules are held in the fieldSpec part of a file. These contain rules challenge user input.
