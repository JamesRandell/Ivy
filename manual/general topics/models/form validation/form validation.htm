The rules in back key of each field match methods defined in the Ivy_Validation file. When input is received these rules are checked for a valid entry.

Rules

Rules have a key followed by a value. Some values must be a specific word where as others can be a generic string or integer.

Type

This matches the type attribute on most RDBMS and should be one of the following

  • var
  • int
  • clob
  • unix

Clob should only be used for free text type fields where Unix tells the framework to parse a Unix time stamp.

Truncate

If the input is longer than the value, the data to the right of this character limit will be discarded.

Options are:

  • a numeric value

Size

This matches the maximum length of data that can be inserted to a database field. If this value is exceeded then an error will be generated.

Options are:

  • a numeric value

Minsize

if the input length is shorter than this value then an error will be generated.

Options are:

  • a numeric value

Unique

A query can be used here to make sure that the input does not exist in the same field/table.

View further information at the bottom for help on structuring this value.

Regex

The validation engine supports regex expressions to filter out undisried input.

Uppercase

Input will be transformed to upper case.

Options are:

  • y

Required

An input is required, if no input is found then an error is generated.

Options are:

  • y

Default

I there is no input found then use a default value.

Options are:

  • Collar - unique user ID
  • Date - current Unix time stamp

If none of the above values are set then whatever else is found will be used as the default value.

Rank

A numeric value can be set so that this field will only be visible to a user with a rank higher than the one you set. This applies only to form fields, the data will be visible on results.
