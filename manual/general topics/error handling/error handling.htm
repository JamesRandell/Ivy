Ivy comes with an error reporting module as part of it's core. Each site that is registered with the framework has every error stored in the database

PHP coding errors

The error module takes a leaf out of Zend Platform and mimics the categories of errors that errors get reported under. Any script error the falls in the standard PHP bitmask is recorded.

Additional error types

Errors that are reported get broken down in to further categories to better identify the generated report.

  • Slow Script Execution
  • Slow Function Execution
  • Slow Query Execution
  • Database Error
  • PHP Error
  • PHP Notice
  • Function Error
  • Excess Memory Usage
  • Load Average
  • Ignore

Triggering errors

To trigger an error in your application you should use the trigger_error function and pass the error string to it.

When this is invoked your error is recorded and can be viewed from the error module.

Help

When viewing the error report you also have links to the originating files, script location and line numbers and also related errors that are of the same type and source.
