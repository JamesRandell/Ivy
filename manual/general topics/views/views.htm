The Views folder contains the templates that draw page elements on the screen.

This includes:

  • Global templates that deals with the overall look and feel of a site from one page to the next
  • Local templates are individual templates assigned to one or more actions. They make the scaffolding for content that sits inside the Global template
  • Global templates Cascading Style sheets
  • Widgets are individual pieces of code that have a single purpose like displaying a result or a form

The Display

All information passed to the display is accessable in a PHP array. To access the information held in this array, the variable $array can be used.

Template engines

Ivy has the capability of supporting multiple template engines (like Smarty or XSL). By default the framework uses it's own - Borne.

Borne syntax is recognised by the pair of curly braces ({}) that surround Borne commands.

It can also handle PHP tags like normal (<?php ?>).

For a list of Borne commands refer to further documentation at the bottom of this page.

Why Borne?

For really simple stuff it is better to learn Borne and use it's own syntax to display your information. For more complex statements you can simply switch to PHP code at will.
