Common practices have been used to keep the code base well organised and structured.

Naming - code

  • Names are in camelCaps with the the first word in lowercase with every first letter of each subsequent word capitalized.
  • Classes are usually seperated with an underscore.
  • Methods only contain A-Z names.
  • Using the word Ivy should be avoided when naming things. It should be treated as a reserved word and is used by the framework.

Naming - file structure

  • Plural of a word are avoided - even if a folder contains multiple files.
  • The folder structure of core, extensions, themes and sites are very similar, with the same names for folders being used throughout.
  • Folder names are kept in lowercase.

Naming - database

  • Plural of a word are avoided - even if a table contains multiple files.
  • All tables, keys, indexes, sequences and fields are in UPPERCASE.
