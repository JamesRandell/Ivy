These blocks of code make up your application. An action is just another name a method in a class that is one of your controllers. Throughout this document the term 'Action' is used - remember that these are just class methods!

URI Routing

An action is called by a parameter set in the URI. When "$_GET['action']" is sent the action is invoked and your code is run.

Non-existing actions

If a call to a non-existing action is made the controller will route the request to the index action (which is present in every controller).

Creating secure actions

Sometimes you will create an action that can only be used by another, and it shouldn't be requested anywhere else.

You can take advantage of PHP5's visibility keywords to declare a the action Private or Protected. Doin'g so will force the controller to route to the index action should a request to a private action be made.
